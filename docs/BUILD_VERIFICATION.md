# Nexora TV — Build Verification

## Purpose

This document records the repeatable Android build verification path for M5 and future milestones.

## CI Build

GitHub Actions workflow:

`Android Build Verification`

Workflow file:

`.github/workflows/android-build.yml`

Primary command:

```bash
gradle :app:assembleDebug --no-daemon --stacktrace
```

Expected output:

- Debug APK artifact: `nexora-tv-debug-apk`
- Build reports artifact: `nexora-tv-build-reports` when reports exist

## Local Build

If Gradle is installed locally:

```bash
gradle :app:assembleDebug --no-daemon --stacktrace
```

If Android Studio is used:

1. Open repository root.
2. Sync Gradle.
3. Select app module.
4. Run `assembleDebug`.

## Gradle Wrapper Status

Gradle wrapper is not added in this patch.

Reason:

- `gradle-wrapper.jar` should be generated by a trusted local Gradle installation.
- This chat environment cannot safely generate and verify the wrapper binary.
- CI uses `gradle/actions/setup-gradle@v4` with Gradle `8.7` instead.

## Runtime Smoke Test

Build verification does not replace runtime QA.

Runtime still requires Android Studio emulator or Android TV / Fire TV device.

Smoke checklist:

1. Launch app.
2. Splash → Login → Activation.
3. Enter `demo123`.
4. Continue to Home.
5. Switch Home / Live TV / Movies / Series / Settings.
6. Select playable mock content.
7. Confirm Detail screen opens.
8. Press Play Mock.
9. Confirm Player opens.
10. Test Back navigation.
