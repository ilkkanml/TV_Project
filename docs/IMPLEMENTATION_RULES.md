# Nexora TV — Implementation Rules

## Core Philosophy

Implementation must prioritize:

- stability
- readability
- maintainability
- premium UX consistency
- controlled complexity

---

# Implementation Rules

## Additive Development

Systems should evolve additively.

Avoid:

- large rewrites
- architecture resets
- uncontrolled dependency replacement

---

## Minimal Dependency Rule

Only add dependencies that are:

- necessary
- stable
- production-safe
- architecture-compatible

Avoid dependency flooding.

---

## TV-First Rule

All UI implementation must prioritize:

- remote navigation
- large readable elements
- TV viewing distance
- cinematic interaction pacing

Not mobile-first interaction models.

---

## Playback Stability Rule

Playback continuity has higher priority than feature quantity.

Avoid:

- unnecessary playback abstraction
- unstable player experimentation
- aggressive feature injection into playback core

---

## UX Consistency Rule

All screens must preserve:

- cinematic feel
- dark premium atmosphere
- smooth transitions
- consistent focus behavior
- calm visual pacing

---

## Complexity Control Rule

Do not introduce:

- unnecessary systems
- enterprise overengineering
- excessive architecture layers
- feature clutter
- unstable experimentation

---

# Final Principle

Nexora TV should feel:

- premium
- stable
- cinematic
- fast
- calm
- modern

Every implementation decision should protect these qualities.
